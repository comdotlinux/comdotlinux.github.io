Experience With Interesting Technologies

Jakarta EE

I am well versed with Jakarta EE (Previously known as Java EE). Since Java EE 5 the framework has transformed into a very productive yet stable platform to develop web applications that not only are secure and extensible but also are able to communicate with wide variety of servies using standard protocols all the while being inherently separated infrastrucure and business code. Further since many competing application servers are available, if we use oly the API's i.e. the standard interfaces then not only is it possible to switch a application server vendor but it is almost painless. Also tried / aware of some new frameworks targeted towards cloud deployments. Some are Quarkus, Micronaut and Helidion

Eclipse Microprofile

Building up on the Jakarta EE platform, Microprofile Specification brings a fast paced (releases 4 times a year) add-ons or tools for the could native microservices. These are Microprofile Config, Metrics, Fault Tolerance and Health Checks to name a few. The cherry on top is these are already implemented by virtually all Major Application server vendors so this adds no extra weight to the Jakarta EE thin war.

Quarkus

Using Microprofile and other proven specs like JAX-RS, Red Hat has created and released a framework that is built to run on cloud. This is Quarkus -- Supersonic Subatomic Java. Using Oracle Labs GraalVM we can AOT compile Quarkus and our business code into a statically linked binary that can onlt only start in less than 100 milliseconds but also consumes less memory and hence can have more replicas serving more requests. But Also the running on the JVM is optimized by compile time optimizations to make it extremely productive.

Docker

Docker revolutionized the packaging and distribution of software by defining esentially a blue print to build the application right from the OS to the code we write. It eliminated the impedence mismatch between Developer workstation and pre-prod / prod environmets. When paired with Jakarta EE or Quarkus We can leverage docker's layered file system to only push / pull the top changed business code and rest is only once.

Git

I read somewhere, (and I agree that it's not an understatement) that Linus Torvalds will be remembered for GIT more than the Linux Kernel. The way Git has revolutionized the version control landspace and combined with Github or Gitlab has compelled even the likes of Microsoft to take notice and start to use it. I am proficient with Git and use it regularly within and out of work, however I am always finding out how Git is way smarter than I realize and there is always something to learn.

Experience With Spring

Used Spring framework extensively along with Spring batch. Spring has become the goto framework for building microservices and the familiarity and semantics along with opinionated but sane defaults make it very attractive and easy to start. Combined with fat jars the running of servies is more or less painless when a JRE is available. I Familiar, comfortable and proficient with Sprint MVC, Spring Data, Sprint Batch and Spring Boot to name a few.

Metrics

Using Prometheus we can collect, store and distribute application metrics to Grafana very easily where we can then plot it using beautiful graphs. Setting it up is very easy and painless and has much greater rewards when queries are properly added to dashboards.

An Implementation of Kubernetes by Red Hat

Once configured, the scaling, monitoring, rolling releases and routing is easily done using Openshift Web Console or the commandline interface, taking the provisoning of environments to the next level.
