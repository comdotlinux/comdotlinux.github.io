Experience With Interesting Technologies

Spring

Used Spring framework extensively along with Spring batch. Spring has become the goto framework for building microservices and the familiarity and semantics along with opinionated but sane defaults make it very attractive and easy to start. Combined with fat jars the running of services is more or less painless when a JRE is available. I am Familiar, comfortable and proficient with the Spring ecosystem.

Gradle

Extremely Good With Gradle, setting up multi-project Builds, Custom Tasks and Plugins To Streamline Routine Development.

Hasura GraphQL & Apollo Kotlin

Using Hasura, Both Cloud and Locally Installed along with Apollo GraphQL Code Generation and for GraphQL Queries, Mutations and Subscriptions.

Jakarta EE & Microprofile

I am well versed with Jakarta EE (Previously known as Java EE). Building up on the Jakarta EE platform, Microprofile Specification brings a fast-paced (releases 4 times a year) add-ons or tools for the could native microservices. These are Microprofile Config, Metrics, Fault Tolerance and Health Checks to name a few. Some are Quarkus, Micronaut and Helidon

Quarkus

Using Microprofile and other proven specs like JAX-RS, Red Hat has created and released a framework that is built to run on cloud. This is Quarkus -- Supersonic Subatomic Java.

Terraform With AWS

Used Terraform, primarily for setting up AWS infrastructure. Familiar with HCL, as well as supporting with maintenance of terraform custom plugin for a cloud provider where terraform module was not available. Used Terraform to setup EKS, RDS, Cloudfront Serving S3, App Runner, Secrets Manager as well all all other related services from AWS

Docker

Docker revolutionized the packaging and distribution of software. Test Containers built upon it to start required software from withing the tests. Test Containers Cloud takes it a step further where you dont have to run the docker containers locally but are run somewhere in the cloud.

Amazon Elastic Kubernetes Service

Proficient with EKS, Deployments, Cron Jobs. Deployment of services using manually created Helm Charts as well as customization and maintenance. Familiar with Deploying Helm Charts from within Terraform using helm provider and using ethe helm CLI.

Github Actions

Able to set up CI with Github Actions for not only Gradle Builds but also Helm Deployments and tests in other languages. Also wrote internal composable actions to enable reuse. Able to setup complex parallel and custom actions with manual triggers as per project requirements.

Metrics
